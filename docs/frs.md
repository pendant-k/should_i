# Should I? - 익명 투표 커뮤니티 플랫폼 기획서

## 📋 프로젝트 개요

### 핵심 컨셉
사용자들이 일상의 크고 작은 결정에 대해 익명으로 투표를 받고, 관심사가 비슷한 사람들의 의견을 참고할 수 있는 모바일 우선 커뮤니티 플랫폼

### 차별화 포인트
- **투표 중심 UX**: 댓글보다 투표가 핵심 (빠른 참여 유도)
- **해시태그 기반 탐색**: 카테고리 + 자유로운 해시태그로 유연한 필터링
- **모바일 최적화**: 스와이프, 원클릭 투표 등 모바일 네이티브 경험
- **익명성 보장**: 솔직한 의견 표현 가능
- **다국어 지원**: 한국어/영어 (향후 확장 가능)

---

## 🎯 타겟 유저

### Primary
- **모바일 중심 사용자**: 이동 중, 잠깐의 여유시간에 빠른 결정 도움 필요
- **우유부단한 성향**: 타인의 의견을 참고하고 싶어하는 사용자
- **Z세대/밀레니얼**: 소셜 미디어에 익숙하고 커뮤니티 참여 적극적

### Use Cases
- 💰 **소비 결정**: "이 옷 살까?", "이 맛집 가볼까?"
- 💼 **커리어 고민**: "이직할까?", "이 프로젝트 맡을까?"
- ❤️ **관계 고민**: "고백할까?", "연락할까?"
- 🎯 **습관 형성**: "헬스장 등록할까?", "강의 들을까?"

## 🏗️ 핵심 기능 구조

### 1. 인증 시스템
```
- 이메일/소셜 로그인 (Google, Kakao)
- 프로필 설정
  └─ 닉네임 (익명성 유지)
  └─ 관심 카테고리 선택 (최대 5개)
  └─ 언어 설정 (한국어/English)
```

### 2. 게시글 작성 (Should I?)
```
[필수]
- 제목: "Should I...?" 형식 (최대 100자)
- 투표 옵션: 2-4개 (Yes/No 또는 커스텀)
- 카테고리 선택 (1개, 대분류)

[선택]
- 상세 설명 (최대 300자)
- 해시태그 (최대 5개) ⭐ 핵심 탐색 기능
- 이미지 첨부 (1장)
- 마감 시간 설정 (24시간/3일/7일/무제한)

💡 해시태그 예시:
#이직 #연봉협상 #스타트업 #대기업
#20대 #30대 (선택적으로 연령대 명시 가능)
#서울 #부산 (지역 태그)
```

### 3. 카테고리 체계 (대분류만)
```
💼 커리어 · Career
💰 돈 · Money
❤️ 관계 · Relationships
🎓 배움 · Learning
🏃 라이프스타일 · Lifestyle
🛍️ 쇼핑 · Shopping
🎨 취미 · Hobbies
💭 일상 · Daily

⭐ 세부 분류는 해시태그로 해결
예: 커리어 카테고리 + #이직 #연봉 #면접 등
```

### 4. 투표 및 인터랙션
```
[투표]
- 원클릭 투표 (재투표 불가)
- 실시간 투표 결과 시각화
- 투표 후 결과 즉시 표시 (참여 보상)

[부가 기능]
- 투표 후 짧은 코멘트 (선택, 50자)
- 북마크/저장
- 공유하기 (투표 결과 이미지 생성)
- 신고 기능
```

### 5. 피드 알고리즘
```
[메인 피드 우선순위]
1. 사용자 관심 카테고리
2. 팔로우한 해시태그
3. 트렌딩 (24시간 내 많은 투표)
4. 마감 임박 게시글
5. 신규 게시글

[탐색 탭]
- 🔥 실시간 인기
- 📁 카테고리별
- ⏰ 마감 임박
- ⚖️ 논쟁중 (투표 비율 45-55%)
- #️⃣ 해시태그 탐색 (트렌딩 태그)
```

### 6. 게임화 요소 (재방문 유도)
```
- 투표 스트릭 (연속 투표 일수)
- 배지 시스템
  └─ "결정 도우미" (100표 달성)
  └─ "현명한 조언자" (내 의견이 다수표된 횟수)
  └─ "트렌드세터" (내 게시글 1000표)
- 레벨 시스템 (활동 점수)
```

### 예상 수익 시뮬레이션 (MAU 50,000 기준)
```
광고 수익:
- 노출당 $0.50 CPM 가정
- 유저당 월 100 노출
- 50,000 × 100 × $0.50 / 1000 = $2,500/월

프리미엄 구독:
- 전환율 2% 가정
- 1,000명 × $2.99 = $2,990/월

총 예상: $5,490/월 ($65,880/년)
```

## 🛠️ 기술 스택 (학습 목적)

### Frontend - Mobile First
- **Next.js 14+** (App Router) - 모바일 최적화 PWA
- **TypeScript**
- **Tailwind CSS** + **shadcn/ui** (모바일 컴포넌트 중심)
- **i18next** (다국어)
- **Zustand** or **Jotai** (상태관리)

- **Framer Motion** (터치 제스처, 스와이프 애니메이션)

- **Touch Gestures**: 스와이프, 롱프레스

- **Pull-to-Refresh**: 모바일 네이티브 경험

### Backend
- **Next.js API Routes** (초기)
- **Prisma** (ORM)
- **PostgreSQL** (데이터베이스)
- **NextAuth.js** (인증)

### Infrastructure
- **Vercel** (호스팅)
- **Vercel Postgres** or **Supabase**
- **Cloudinary** (이미지 저장)
- **Google Analytics**

### 추가 고려사항
- **Redis** (캐싱, 실시간 투표 카운트)
- **Websocket** (실시간 투표 업데이트)

## 🎨 UI/UX 핵심 원칙

### 1. 빠른 투표 경험 (Mobile Optimized)
```
- 메인 화면에서 바로 투표 가능
- 스와이프로 다음 게시글 (Tinder-like)
- 원클릭 투표 → 즉시 결과 표시 (햅틱 피드백)
- Pull-to-refresh로 새 게시글
```

### 2. 깔끔한 정보 계층 (모바일 카드)
```
[게시글 카드]
┌─────────────────────────────────┐
│ 💼 커리어 · 2시간 전               │
│ #이직 #연봉 #스타트업              │
│                                   │
│ Should I quit my job and          │
│ start freelancing?                │
│                                   │
│ ⭕ Yes, go for it! (234) 68%      │
│ ⭕ Not yet (112) 32%              │
│                                   │
│ 💬 12  🔖 Save  ⚡ 346 votes      │
└─────────────────────────────────┘

👆 탭으로 투표
👈 스와이프로 다음
```

### 3. 제스처 기반 인터랙션
```
- 스와이프 업: 다음 게시글

```

### 4. 다크모드 우선 설계
- 장시간 사용을 고려한 눈의 피로 감소
- 배터리 절약 (OLED 디스플레이)

### 5. 해시태그 중심 디자인
- 게시글 상단에 해시태그 prominence
- 탭 한 번에 해시태그 필터링

## 🔧 해시태그 시스템 설계

### 데이터베이스 스키마
```prisma
model Post {
  id          String   @id @default(cuid())
  title       String
  category    Category
  hashtags    Hashtag[]
  // ...
}

model Hashtag {
  id          String   @id @default(cuid())
  name        String   @unique
  posts       Post[]
  useCount    Int      @default(0)
  trendScore  Float    @default(0)
  createdAt   DateTime @default(now())
}
```

### 트렌딩 알고리즘
```typescript
// 트렌딩 점수 = (최근 24시간 사용 횟수) * 시간 감쇠 계수
trendScore = (recentUseCount * 10) / (hoursSinceFirstUse + 2)
```

### 자동완성
- 사용자가 '#' 입력 시 인기 해시태그 제안
- 최근 사용한 해시태그 우선 표시
- 오타 방지 (fuzzy matching)
